# StackCoin Python SDK Redesign

> **Goal:** Replace the verbose auto-generated `stackcoin-python` client and LuckyPot's untyped `stk.py` with a single ergonomic Python SDK that combines generated Pydantic v2 models with a handwritten REST client and WebSocket gateway.

**Date:** 2026-03-05

---

## Problem

Two Python layers exist for talking to the StackCoin API, and both have significant gaps:

1. **`tmp/stackcoin-python/`** -- Auto-generated via `openapi-python-client`. Full type information (34 model files, 15 API function files), but extremely verbose usage (`await stackcoin_send_stk.asyncio(client=client, user_id=id, body=SendStkParams(amount=100))`). No WebSocket gateway support.

2. **`tmp/LuckyPot/luckypot/stk.py`** -- Handwritten httpx wrapper. Clean API (`await stk.send_stk(user_id, 100)`), but every return is `dict | None`. No type safety, no autocomplete, typos in dict key access are silent runtime errors.

## Solution

A single `stackcoin` package with three layers:

| Layer | Source | Purpose |
|-------|--------|---------|
| **models.py** | Generated by `datamodel-codegen` | Pydantic v2 models for all API schemas |
| **client.py** | Handwritten | Ergonomic async REST client |
| **gateway.py** | Handwritten (from LuckyPot's gateway.py) | WebSocket event gateway |
| **errors.py** | Handwritten | Single `StackCoinError` exception |

## Package Structure

```
tmp/stackcoin-python/
├── justfile                     # generate models, format, install
├── pyproject.toml               # package metadata + deps
└── stackcoin/
    ├── __init__.py              # re-exports: Client, Gateway, Event, StackCoinError
    ├── models.py                # GENERATED -- never hand-edit
    ├── client.py                # handwritten REST client (~150 lines)
    ├── gateway.py               # handwritten WebSocket gateway (~130 lines)
    └── errors.py                # StackCoinError (~15 lines)
```

Dependencies: `httpx`, `websockets`, `pydantic>=2`

## Client API

```python
import stackcoin

async with stackcoin.Client(base_url="http://localhost:4000", token="...") as client:
    # User lookups
    me = await client.get_me()                              # -> User
    user = await client.get_user(user_id=42)                # -> User
    users = await client.get_users(discord_id="123456789")  # -> list[User]

    # Transfers
    result = await client.send(
        to_user_id=42, amount=100, label="winnings",
        idempotency_key="pot_win:1:42"
    )  # -> SendStkResponse
    # result.transaction_id, result.from_new_balance -- typed, autocomplete

    # Payment requests
    req = await client.create_request(
        to_user_id=42, amount=5, label="Pot entry",
        idempotency_key="pot_entry:1:123"
    )  # -> CreateRequestResponse
    # req.request_id, req.status -- typed

    request = await client.get_request(request_id=789)  # -> Request
    await client.accept_request(request_id=789)          # -> RequestActionResponse
    await client.deny_request(request_id=789)            # -> RequestActionResponse

    # Transactions
    txns = await client.get_transactions()               # -> list[Transaction]
    txn = await client.get_transaction(transaction_id=5) # -> Transaction

    # Events (REST fallback)
    events = await client.get_events(since_id=0)         # -> list[dict]

    # Discord guild info
    guilds = await client.get_discord_guilds()            # -> list[DiscordGuild]
    guild = await client.get_discord_guild("123456")      # -> DiscordGuild
```

### Design Principles

- Methods are verbs matching intent, not URL paths
- Parameters are simple Python args, not wrapper objects
- Returns are typed Pydantic models
- `idempotency_key` is an optional kwarg on mutating methods
- Long-lived client with connection pooling (httpx.AsyncClient reuse)
- Async context manager for cleanup

## Gateway API

```python
gateway = stackcoin.Gateway(
    ws_url="ws://localhost:4000/bot/websocket",
    token="...",
    last_event_id=0,
)

@gateway.on("request.accepted")
async def handle_accepted(event: stackcoin.Event):
    print(event.type)              # "request.accepted"
    print(event.id)                # 42
    print(event.data["request_id"])

await gateway.connect()  # blocks, auto-reconnects
```

`Event` model:
```python
class Event(BaseModel):
    id: int
    type: str
    data: dict
    inserted_at: str
```

Gateway is a separate class from Client -- they share credentials but are independent. You can use REST without gateway, or gateway without REST.

## Error Handling

```python
class StackCoinError(Exception):
    status_code: int    # HTTP status code
    error: str          # e.g. "insufficient_balance", "not_found"
    message: str | None # human-readable detail

try:
    await client.send(to_user_id=42, amount=999999)
except stackcoin.StackCoinError as e:
    print(e.status_code)  # 400
    print(e.error)        # "insufficient_balance"
```

Single exception class. Callers check `e.error` string if they need to distinguish error types.

## Model Generation

Models are generated from the existing `openapi.json` using `datamodel-codegen`:

```bash
datamodel-codegen \
  --input ../../openapi.json \
  --input-file-type openapi \
  --output-model-type pydantic_v2.BaseModel \
  --output stackcoin/models.py
```

The `justfile` automates this. Models are never hand-edited.

## LuckyPot Migration

After the library is built:

1. **`stk.py`** becomes a thin module exposing a `stackcoin.Client` singleton
2. **`game.py`** switches from `dict | None` returns to typed models + exceptions
3. **`gateway.py`** is deleted -- replaced by `stackcoin.Gateway`
4. **`pyproject.toml`** adds `stackcoin` as a dependency (editable local install)
