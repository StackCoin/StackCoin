# `defevent` Macro Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace triple-defined event schemas (Ecto, OpenApiSpex, Python) with a single `defevent` macro that generates both Ecto embedded schemas (for internal validation) and OpenApiSpex schemas (for spec generation) from one canonical definition.

**Architecture:** A `StackCoin.EventSchema` module provides the `defevent` macro. `StackCoin.Core.EventData` uses it to define all event types. The macro generates Ecto embedded schema submodules, OpenApiSpex data+event wrapper schemas, and a registry. A `__before_compile__` callback auto-generates the `Event` discriminated union and `EventsResponse` schemas. `Event.create_event/3` is updated to validate data through the Ecto changeset before persisting.

**Tech Stack:** Elixir macros, Ecto embedded schemas, OpenApiSpex

---

### Task 1: Create the `StackCoin.EventSchema` macro module

**Files:**
- Create: `lib/stackcoin/event_schema.ex`

The macro module provides `defevent` and `__before_compile__`. Each `defevent` call:
1. Parses the field definitions from the block
2. Defines an Ecto embedded schema submodule with changeset
3. Defines an OpenApiSpex data schema module under `StackCoinWeb.Schemas`
4. Defines an OpenApiSpex event wrapper schema module under `StackCoinWeb.Schemas`
5. Registers the event type in `@events` accumulator

`__before_compile__` generates:
- `StackCoinWeb.Schemas.Event` (discriminated union)
- `StackCoinWeb.Schemas.EventsResponse` (list wrapper)
- `schema_for/1` function
- `event_types/0` function
